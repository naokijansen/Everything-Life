01 Concept & identity concept ▶
OpenClaw identity — defined the character brief: brutally honest, no-filter accountability partner with a professional edge. Distinct from the personal assistant agent.
Agent architecture — designed a 2-agent system. Focused on Agent 1 (PA). Full feature set mapped: quadrant classification, escalation rules, quadrant drift detection, adaptive 3-ping daily rhythm based on wake time, Telegram interface on VPS.
Productivity matrix defined — 4 quadrants: 🔴 Red (want to do, counterproductive), 🟢 Green (want to do, productive), 🟡 Yellow (don't want to do, pointless), 🔵 Blue (don't want to do, necessary).
Design decisions — soft dark mode (warm grays, Claude-style), quadrant as main view, done stack on right, Calendar and Heatmap tabs, GitHub-style heatmap, JSON backend on VPS, Cloudflare subdomain, no auth, responsive for laptop and phone.
02 Frontend MVP frontend ▶
Single HTML file — 4 interactive quadrant panels with color-coded tasks.
Task interactions — add tasks per quadrant; check off tasks → animate into done stack with timestamp and color dot; right-click done items → context menu to move back to any quadrant.
Drag and drop — drag tasks between quadrants with visual drop highlighting.
Heatmap v1 — full year GitHub-style heatmap, green intensity scaling by task count, click day → popup with every task completed that day, today highlighted with green ring.
Archive button — pushes done tasks into heatmap history.
03 Backend & deployment backend ▶
REST API (Node.js/Express) — three endpoints: read state, write state, archive done tasks. All protected with x-api-key header. State persisted to state.json. Frontend served as static files from public/.
Frontend wired to API — loads state on boot, saves after every change with 600ms debounce, loading screen on boot, sync dot in header (grey/yellow/green/red) showing save status. Archive button calls API instead of only updating local state.
Infrastructure — Node.js via nvm on Ubuntu VPS, PM2 for process persistence, Nginx as reverse proxy (port 80/443 → Node 3001), SSL via Let's Encrypt/certbot, Cloudflare DNS A record → sankyunao.dev, ports 80 and 443 opened in ufw and Hetzner firewall, Cloudflare SSL mode set to Full.
Midnight cron job — auto-archives any uncleared done tasks into heatmap history at midnight.
Deployment fix — corrected filename from index_1.html → index.html on server via mv.
04 Calendar tab — built from scratch calendar ▶
Architecture — calendar events are independent from quadrant tasks, stored in state.calendar key, persisted to state.json. Added calendar: {} and calNextId: 1 to DEFAULT_STATE; validateState updated for backward-compatible migration.
Week view — time grid, colored event blocks, current-time red line, sticky day headers, all-day strip, month view toggle. Prev/next/today navigation with dynamic heading.
Event creation — click empty time slot or + Add button → modal for name, date, from/to time, category. Click any event to edit.
Categories v1 — Class (blue), Birthday (purple), Personal (yellow), Work (green), Study (red), Hobby (orange), Chore (dark purple).

Drag to move — pointer events (touch support), cursor-offset fix (no jump when grabbing mid-event), 15-minute snapping, cross-day support.
Drag to resize — bottom edge handle, 15-minute snapping.
Overlap layout — greedy column-track algorithm so concurrent events sit side by side instead of stacking.
Arrow key navigation — left/right to move between weeks/months. Hover an event + up/down to nudge ±15min, debounced save.
All-day events — checkbox in modal, sticky strip above time grid that doesn't scroll.
Cmd+Z undo — per-action undo stack (add, edit, move, resize, delete) plus bulk undo for imports.
iCal import — .ics parser with UTC timezone handling, TZID detection, weekly recurring event expansion with BYDAY support, keyword-based category guessing.
Month view — Fantastical-style grid, colored pills per day, click to jump to that week.

08 Bug fixes fixes ▶
Drag jump bug — double-counted scrollTop because getBoundingClientRect() already accounts for scroll.
iCal timezone — events were 2 hours early; parser stripped the Z suffix and treated UTC times as local. Fixed by passing Z through to new Date() so browser converts correctly.
Hour density — reduced from 64px to 44px per hour so the full day is visible with less scrolling.
Modal overflow — widened and added box-sizing: border-box so date/time fields don't bleed outside the card.
Dropdown styling — appearance: none with custom SVG chevron, color-scheme: dark on native date/time inputs.
Heatmap mobile scroll — was auto-scrolling to scrollWidth (far right = oldest). Fixed by switching to 16-week render that fits the screen without needing scroll.
09 Data integrity & backup system infra ▶
Heatmap data loss fix — the 600ms debounced saveState() could fire mid-archive and POST old state back, overwriting what /api/archive had just written. Fix: cancel pending save timer at the start of clearDone(), then explicitly call saveState() after local state is updated. One April 5th entry (4 tasks) was confirmed lost and unrecoverable.
Rolling backup system — every /api/archive call snapshots state.json to ~/openclaw/backups/state-YYYY-MM-DD.json before mutating. First daily POST /api/state also writes a backup. Files older than 30 days are auto-pruned. GET /api/backups lists what's available. Restoring is one cp command.
PM2 PATH fix — fixed pm2 PATH issue for SSH deploys. Added pm2pi as a persistent shell alias on Mac.
